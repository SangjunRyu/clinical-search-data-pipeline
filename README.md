## 📁 디렉터리 구조 및 역할 설명

본 프로젝트는 **물리적인 서버 기준이 아닌, 역할(Responsibility) 기준으로 디렉터리를 분리**하여  
확장성과 가독성을 고려한 데이터 엔지니어링 파이프라인 구조를 따릅니다.

---

### 📥 ingestion/ – 데이터 수집(Ingestion) 레이어

`ingestion` 디렉터리는 **원천 데이터를 수집하여 Kafka로 전달하는 역할**을 담당합니다.  
Kafka Producer 실행 코드, 설정 파일, 공통 유틸리티 및 실행 로그를 포함합니다.

#### ▪ `producer/`
- Kafka Producer 실행 코드가 위치한 디렉터리
- 파일 기반 로그 데이터를 읽어 Kafka 토픽으로 전송
- 대량 데이터 전송을 고려한 배치 지향 구조
- 소형 인스턴스 환경에서도 안정적으로 동작하도록 비동기 전송 및 주기 로그 방식 적용

#### ▪ `config/`
- Kafka 브로커 정보 등 실행 환경에 필요한 설정 파일 관리
- 실제 설정 파일(`config.yaml`)은 보안 및 환경 분리를 위해 Git 추적 대상에서 제외
- 예시 파일(`config.yaml.example`)을 통해 실행 방법 안내

#### ▪ `utils/`
- 여러 모듈에서 공통으로 사용하는 유틸리티 코드
- 파일 기반 로깅 설정(`logging.py`)을 포함하여 stdout 출력에 따른 성능 저하 방지
- 대량 전송 시에도 장애 분석이 가능하도록 로그 구조 표준화

#### ▪ `logs/`
- Kafka Producer 실행 중 생성되는 런타임 로그 저장 디렉터리
- Git 추적 대상에서 제외되며, 장애 발생 시 원인 분석을 위한 로그 보관 목적

---

### 📡 messaging/ – 메시징 인프라 레이어

`messaging` 디렉터리는 **Kafka 브로커 및 메시징 인프라 구성**을 담당합니다.

- Kafka 브로커 실행을 위한 Docker / Docker Compose 설정 파일 포함
- 애플리케이션 코드와 인프라 설정을 분리하여 관리
- 향후 멀티 브로커 구성 또는 Kubernetes 환경으로 확장 가능하도록 설계
